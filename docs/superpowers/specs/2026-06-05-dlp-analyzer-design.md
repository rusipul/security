# DLP 보안 위협 분석기 설계 문서

**작성일:** 2026-06-05  
**프로젝트:** 오피스키퍼 DLP 월별 리포트 자동 분석 도구  
**대상 파일:** 오피스키퍼_정보유출방지_[회사명]_YYYYMM.xlsx

---

## 1. 개요

회사 보안 담당자가 매월 수령하는 오피스키퍼 DLP Excel 리포트를 자동으로 분석해, 보안 위협 요소를 탐지하고 한 장짜리 요약 보고서를 생성하는 데스크톱 GUI 애플리케이션.

**핵심 목표:**
- 7만 건 이상의 로그에서 위협 이벤트를 자동 추출
- 즉시 조치가 필요한 사용자와 행위를 식별
- Claude AI가 자연어 위협 요약 및 권고사항 생성

---

## 2. 기술 스택

| 항목 | 선택 |
|------|------|
| 언어 | Python 3.11+ |
| GUI | CustomTkinter |
| 데이터 분석 | pandas, openpyxl |
| 보고서 생성 | openpyxl |
| AI 요약 | Claude API (claude-sonnet-4-6) |
| 배포 | PyInstaller (.exe 단일 파일) |

## 5. 분석 엔진 — 탐지 규칙 7가지

### 규칙 1: USB 시도자 🔴 고위험
- **시트:** `매체제어 로그_매체현황`
- **조건:** DLP 차단 여부와 무관하게 USB 연결 시도한 사용자 전원
- **출력:** 사용자명, 부서, 장치 정보, 시도 횟수, 시도 일시

### 규칙 2: 설계 압축파일 첨부 🔴 고위험
- **시트:** `첨부파일 로그_매체현황`, `첨부파일 로그_보안인증 확인현황`
- **조건:** 파일 확장자가 `.tar`, `.gz`, `.tg` 인 첨부파일
- **출력:** 사용자명, 파일명, 파일 경로, 용량, 발송 시각

### 규칙 3: 설계 키워드 파일 첨부 🔴 고위험
- **시트:** `첨부파일 로그_매체현황`, `첨부파일 로그_보안인증 확인현황`
- **조건:** 파일명에 `DB`, `SCH`, `REV` 등 설계 관련 키워드 포함
- **키워드 관리:** `keywords.json`의 `design_keywords` 항목에서 추가·수정 가능
- **출력:** 사용자명, 파일명, 매칭 키워드, 용량, 발송 시각

### 규칙 4: 대용량 ZIP 첨부 🟠 주의
- **시트:** `첨부파일 로그_매체현황`, `첨부파일 로그_보안인증 확인현황`
- **조건:** 파일 확장자 `.zip` AND 용량 10MB 이상
- **기준값:** `keywords.json`의 `zip_size_threshold_mb` 항목에서 변경 가능
- **출력:** 사용자명, 파일명, 용량, 발송 시각

### 규칙 5: AI 플랫폼 파일 업로드 🔴/🟠
- **시트:** `인터넷 제어 외부낙뢰`, `첨부파일 로그_매체현황`
- **조건:** 아래 도메인 접근 또는 파일 업로드 탐지
  - `chat.openai.com`, `chatgpt.com` (ChatGPT)
  - `gemini.google.com` (Gemini)
  - `claude.ai` (Claude)
  - `copilot.microsoft.com` (Copilot)
  - `perplexity.ai` (Perplexity)
  - `notebooklm.google.com` (NotebookLM)
- **업로드 판별:** `첨부파일 로그_매체현황`에서 Chrome·Edge·Firefox 등 브라우저 프로세스로 파일이 기록되고, 동일 사용자가 같은 날 AI 도메인 접속 기록이 있으면 업로드로 판정 → 🔴. AI 도메인 접속만 있고 첨부파일 로그에 없으면 단순 접속 → 🟠
- **도메인 관리:** `keywords.json`의 `ai_domains` 항목에서 추가·수정 가능
- **출력:** 사용자명, 접속 사이트, 파일 업로드 여부, 접속 횟수, 일시

### 규칙 6: 외부 이메일 첨부 차단 시도 🔴 고위험
- **시트:** `인터넷 제어 외부낙뢰`
- **조건:** Naver메일, Gmail, Daum메일 등 외부 이메일 서비스 접근 차단 기록
  - `mail.naver.com`, `mail.google.com`, `mail.daum.net` 등
- **도메인 관리:** `keywords.json`의 `external_mail_domains` 항목에서 추가·수정 가능
- **출력:** 사용자명, 시도 서비스, 차단 횟수, 최근 시도 일시

### 규칙 7: WeChat/KakaoTalk 파일 전송 빈도 🟠 주의
- **시트:** `첨부파일 로그_매체현황` (WeChat/KakaoTalk 경로 필터)
- **조건:** WeChat 또는 KakaoTalk을 통해 파일 전송한 이력 전체
- **출력:** 사용자명, 플랫폼, 파일명, 파일 경로, 전송 횟수, 총 용량, 기간

## 6. keywords.json 구조

```json
{
  "design_keywords": ["DB", "SCH", "REV", "PCB", "BOM", "GERBER"],
  "design_extensions": [".tar", ".gz", ".tg", ".brd", ".sch"],
  "zip_size_threshold_mb": 10,
  "ai_domains": [
    "chat.openai.com",
    "chatgpt.com",
    "gemini.google.com",
    "claude.ai",
    "copilot.microsoft.com",
    "perplexity.ai",
    "notebooklm.google.com"
  ],
  "external_mail_domains": [
    "mail.naver.com",
    "mail.google.com",
    "mail.daum.net",
    "mail.yahoo.com"
  ]
}
```

## 8. AI 요약 (Claude API)

분석 완료 후 탐지 결과 요약을 Claude API에 전달해 다음 항목 생성:

1. **이달의 위협 총평** (2~3문장)
2. **상위 3명 사용자 행동 패턴 코멘트** — 각 1~2문장
3. **즉시 조치 권고사항** — 불릿 포인트

모델: `claude-sonnet-4-6`  
결과는 요약 시트 하단에 삽입.

## 9. 실행 흐름

```
1. GUI 실행
2. Excel 파일 선택 (드래그 or 버튼)
3. 설정 확인 (사용자 수, API 키, 키워드)
4. [분석 시작] 클릭
5. analyzer.py → 19개 시트 로딩 및 7개 규칙 적용
6. Claude API → AI 요약 생성
7. reporter.py → Excel 보고서 생성 (reports/ 폴더)
8. GUI에서 완료 알림 → [보고서 열기] 버튼
```
